Popis projektu:

DOMINATOR是一種設備-用於氣槍，彩彈射擊等運動的秒錶。 秒錶使用Arduino控制微控制器（Arduino Uno / Nano）或單獨的芯片（Atmel）AtMega328P，它控制整個系統邏輯。 該秒錶包括一個LCD字符顯示屏，其尺寸為16x2（16個字符x 2行）或20x4，在其上顯示各個團隊的時間。 遊戲中有2個團隊- RED隊 和 GRE隊 互相作用並佔據該設備-DOMINATOR形成的虛點。 在遊戲開始時，顯示屏顯示 RED隊 ， GRE小組 。 團隊中的每個成員都配備有卡，手鐲或密鑰卡，如果在範圍內，它們可以與RFID讀取器以13.56MHz的頻率進行通信。 鑰匙鍊和手鐲可以用顏色編碼（根據團隊和功能，用綠色，紅色表示），從而可以更有效地將其分配給玩家，也可以對其進行存檔和存儲。 如果GRE小組佔據了一個分數，並且該小組的成員將卡附在讀取器上，蜂鳴器會再次響起，該小組的LED會亮起，RED的小組時間會暫停，並且團隊時間很重要GRE佔據了重點。 如果組織者（裁判員）附加了卡片/密鑰卡，則兩次暫停，直到其中一個團隊插入卡片為止。 因此，這種類型的卡牌適合開始休息，但也可以在遊戲結束時需要停止評估遊戲的時間時使用。 在DOMINATOR中還有第四種類型的卡-橡皮擦，在啟動DOMINATOR秒錶後，這兩種卡都會暫停兩次並重置為初始值。 用來重新啟動遊戲，初始化新遊戲。 該項目適用於可以建造設備的Airsoft活動，體育設施或Airsoft團隊。 DOMINATOR系統的優點是可以克隆該系統，並通過機器代碼以相同的配置創建X點，這些代碼可以在遊戲中獨立進行，而RFID卡將在所有DOMINATOR設備上完全起作用。 該系統經過測試，功能齊全，易於維護。 恩智浦RC522閱讀器的工作頻率為13.56MHz，註冊RFID標籤的距離約為3厘米（需要與閱讀器進行幾乎物理接觸），兼容ISO / IEC 14443-A標籤格式。 給定的標籤標準還包括ISIC，充當NFC卡並可以將其物理地址-UID傳達給范圍內的讀取器的總線卡。

項目硬件-RFID統治者：

  • Arduino Uno/Nano
  • RFID 讀者 RC522 (13.56MHz)

  • 蜂鳴器
  • 2x 發光二極管

    • 硬件方面的功能原理：

  • RC522 RFID閱讀器通過SPI總線與Arduino通信。 它讀取範圍內的RFID卡的UID（標識符），將此標識符轉發給Arduino，後者執行程序邏輯。 讀取器對芯片距離天線約3 cm的距離作出響應。
  • Arduino在多種情況下比較卡的讀取地址，將數字狀態應用於二極管，每100毫秒（十分之一秒的計數）運行一個通用計數器，控制兩次計數。
  • 時間通過I2C總線發送到20x4字符顯示器，在此處進行繪製。 在100 kHz的I2C總線速度-標準速度下，顯示可持續長達30ms。
    • 可以在系統中列出的最大時間範圍： 99小時59分59秒
    消耗：5V電源時約110mAh。 適用於通過移動電源，帶有USB輸出的手機的充電器供電。

    /*|---------------------------------------------|*/
/*|標題：加載UID RFID卡                          |*/
/*|說明：通過RFID讀取器RC522讀取                  |*/
/*|粘貼並打印的RFID卡的地址                       |*/
/*|UART接口上的UID。                             |*/
/*|記錄的UID卡用於DOMINATOR                      |*/
/*|作者: Martin Chlebovec                       |*/
/*|E-mail (English only): martinius96@gmail.com |*/
/*|使用許可證: MIT                               |*/
/*|修訂版: 12. August 2020                       |*/
/*|---------------------------------------------|*/

#include <SPI.h>
#include <MFRC522.h>
#define RST_PIN         9
#define SS_1_PIN        10
#define NR_OF_READERS   1
byte ssPins[] = {SS_1_PIN};
MFRC522 mfrc522[NR_OF_READERS];
void setup() {
  Serial.begin(9600);
  while (!Serial);
  SPI.begin();
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    mfrc522[reader].PCD_Init(ssPins[reader], RST_PIN);
    mfrc522[reader].PCD_SetAntennaGain(112); // set to max (00001110)
    Serial.print(F("Reader "));
    Serial.print(reader);
    Serial.print(F(": "));
    mfrc522[reader].PCD_DumpVersionToSerial();
  }
}

void loop() {
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    if (mfrc522[reader].PICC_IsNewCardPresent() && mfrc522[reader].PICC_ReadCardSerial()) {
      Serial.print(F("Reader "));
      Serial.print(reader);
      Serial.print(F(": Card UID:"));
      dump_byte_array(mfrc522[reader].uid.uidByte, mfrc522[reader].uid.size);
      Serial.println();
      Serial.print(F("PICC type: "));
      MFRC522::PICC_Type piccType = mfrc522[reader].PICC_GetType(mfrc522[reader].uid.sak);
      Serial.println(mfrc522[reader].PICC_GetTypeName(piccType));
      mfrc522[reader].PICC_HaltA();
      mfrc522[reader].PCD_StopCrypto1();
    }
  }
}

void dump_byte_array(byte *buffer, byte bufferSize) {
  for (byte i = 0; i < bufferSize; i++) {
  }
  Serial.print("Code for DOMINATOR: ");
  unsigned long kod = 10000 * buffer[4] + 1000 * buffer[3] + 100 * buffer[2] + 10 * buffer[1] + buffer[0]; //finalny kod karty
  Serial.println(kod);
}

    I2C掃描儀-用於讀取LCD顯示屏的I2C轉換器地址的程序

    I2C掃描儀的預期輸出：

    Wire庫內置在Arduino IDE中，無需安裝。 該程序可以立即上傳。 建議在啟動程序之前連接硬件-用於16x2 / 20x4顯示屏的I2C轉換器，即使沒有顯示屏也是如此。 
    #include <Wire.h>
 
 
void setup()
{
  Wire.begin();
 
  Serial.begin(9600);
  while (!Serial);             // Leonardo: wait for serial monitor
  Serial.println("\nI2C Scanner");
}
 
 
void loop()
{
  byte error, address;
  int nDevices;
 
  Serial.println("Scanning...");
 
  nDevices = 0;
  for(address = 1; address < 127; address++ )
  {
    // The i2c_scanner uses the return value of
    // the Write.endTransmisstion to see if
    // a device did acknowledge to the address.
    Wire.beginTransmission(address);
    error = Wire.endTransmission();
 
    if (error == 0)
    {
      Serial.print("I2C device found at address 0x");
      if (address < 16)
        Serial.print("0");
      Serial.print(address,HEX);
      Serial.println("  !");
 
      nDevices++;
    }
    else if (error==4)
    {
      Serial.print("Unknown error at address 0x");
      if (address < 16)
        Serial.print("0");
      Serial.println(address,HEX);
    }    
  }
  if (nDevices == 0)
    Serial.println("No I2C devices found\n");
  else
    Serial.println("done\n");
 
  delay(5000);           // wait 5 seconds for next scan
}

    Button Dominator-氣槍秒錶 彩彈射擊

    Button DOMINATOR是一種設備-用於氣槍，彩彈射擊等運動的秒錶。 秒錶使用Arduino控制微控制器（Arduino Uno / Nano）或單獨的芯片（Atmel）AtMega328P，它控制整個系統邏輯。 該秒錶包括一個LCD字符顯示屏，其尺寸為16x2（16個字符x 2行）或20x4，在其上顯示各個團隊的時間。 遊戲中有2個團隊- RED隊 和 GRE隊 互相作用並佔據該設備-DOMINATOR形成的虛點。 在遊戲開始時，顯示屏顯示 RED隊 ， GRE小組 。 按鈕DOMINATOR使用按鈕輸入來執行團隊成員佔據點的動作，以及輸入裁判和橡皮擦實體。 因此，總共使用了4個系統實體的4個按鈕，即以INPUT_PULLUP模式連接的切換按鈕。 按下按鈕會將GND信號饋送到Arduino數字輸入端子（低電平有效開關信號）。 使用的是切換按鈕，而不是切換按鈕（它們不會對系統邏輯做出反應，所有軟件都將始終被“按下”）。 在基本版本中，DOMINATOR會立即對新聞作出反應-給定團隊的計時被激活（新聞長度，新聞次數等未得到驗證...）。 時間顯示在16x2或20x4的LCD字符顯示屏上，該顯示屏通過I2C總線與控制微控制器通信。 如果RED團隊的成員通過按一個按鈕佔據一個點，則會觸發蜂鳴器，並發出一聲短促的嗶嗶聲來宣布該點的變化。 然後RED小組在此站點上的LED會亮起，並且顯示屏將開始為該小組增加時間。 如果該點已被GRE團隊佔用，並且團隊成員按下相應的按鈕，則蜂鳴器會再次響起，GRE團隊LED會亮起，RED團隊時間會暫停，並且GRE小組的時間是根據他佔據的時間來計算的。 如果組織者（裁判員）按下他的按鈕，則兩個時間都將暫停，直到其中一個團隊的成員進行用戶輸入為止。 因此，這種類型的輸入適合開始休息，但也可以結束遊戲進行評估。 Button DOMINATOR具有用於橡皮擦實體的第四個按鈕，該按鈕在啟動Button DOMINATOR秒錶後會暫停並將兩個時間都重置為初始值。 用來重新啟動遊戲，初始化新遊戲。 該項目適用於可以建造設備的Airsoft活動，體育設施或Airsoft團隊。 Button DOMINATOR系統的優點是可以通過機器代碼克隆並創建具有相同配置的X點，可以將其加載到無限數量的Arduino開發板和芯片中，並保證相同的應用程序操作。 因此，玩家可以在遊戲中獲得更多的積分，而在遊戲結束時，只要哪個得分被哪個團隊佔用，時間就會累加。
