Vorsicht vor RFID-Lesegeräten RC522 mit gefälschtem (nicht originalem NXP RC522) Chip! - Im Projekt auf dem UART wird die Chipversion 0x12 - gefälscht gemeldet (funktioniert nicht für das DOMINATOR-Projekt, der ursprüngliche NXP-Chip, mit dem die Bibliothek zusammenarbeitet, ist erforderlich!)

    • Projektbeschreibung:

  • Gerät - DOMINATOR enthält einen Steuer-Mikrocontroller (Arduino Uno, Nano oder Mega 2560).
  • Die LCD-Zeichenanzeige zeigt die Zeiten der einzelnen Teams (RF- und GDP-Team bzw. BLU- und RED-Team) an.
  • In der Basisversion der Software gibt es 2 Teams, die gegeneinander spielen und diesen bestimmten Punkt belegen!
  • Zu Beginn des Spiels zeigt das Display die Zeiten 00 Stunden, 00 Minuten, 00 Sekunden für Team 1 (ROT), Team 2 (BLAU) an.
  • Jedes Mitglied des Teams ist mit einer Karte oder einem Schlüsselanhänger ausgestattet, die mit einem RFID-Lesegerät auf der Frequenz 13,56 MHz kommunizieren.
  • Schlüsselanhänger können farbcodiert sein (blau, rot nach Team und Funktion ...)
  • Wenn ein Team einen Punkt erzielt, hängt jeder Spieler dieses Teams seine Karte an den Leser an und vergibt dabei den Punkt.
  • Die Diode des gegebenen Teams (rot / blau) leuchtet an dieser Station auf und die Zeit wird dem gegebenen Team hinzugefügt (die Änderung kann auch durch einen Summer angekündigt werden - einen Detektor, falls er im System verwendet wird).
  • Wenn ein Punkt vom anderen Team besetzt ist und ein Mitglied dieses Teams seine Karte anbringt, leuchtet die LED dieses Teams auf, die Zeit des anderen Teams wird angehalten und die Zeit des Teams, das den Punkt besetzt hat, wird gezählt.
  • Wenn die Karte / der Schlüsselbund vom Veranstalter (Schiedsrichter) angebracht wird, werden beide Male angehalten, bis eines der Teams die Karte einführt.
  • Diese Art von Karte eignet sich auch zum Beenden des Spiels, wenn die Zeit stehen bleibt und der Gewinner ermittelt wird.
  • Die letzte mögliche Karte im System ist die Verwendung der sogenannten Radiergummikarte, die beide Male angehalten und zurückgesetzt wird.
  • Das Projekt eignet sich für Airsoft-Veranstaltungen und Sportanlagen.
  • Das System ist getestet, funktionsfähig, funktioniert ohne Probleme und kann die Zeit in Zehntelsekunden messen und anzeigen (programmgesteuert, aber nicht angezeigt, Anzeige in Sekunden).
  • Der RC522-Leser arbeitet mit einer Frequenz von 13,56 MHz, registriert RFID-Tags bei etwa 3 Zentimetern (fast physischer Kontakt mit dem Leser ist erforderlich), kompatibles ISO / IEC 14443 A-Tag-Format.

    • Hardware für das Projekt - RFID DOMINATOR:

  • Arduino Uno/Nano
  • RFID Leser RC522 (13.56MHz)
  • ISO / IEC 14443-A Standard-Schlüsselanhänger / Karten / Armbänder
  • Detektor (passiver Summer)
  • 2x LED diodes

    • Prinzip der Hardwarefunktionalität:

  • Der RF5-Leser RC522 kommuniziert mit dem Arduino über den SPI-Bus. Es liest die UID (Kennung) von RFID-Karten in Reichweite und leitet diese Kennung an Arduino weiter, das die Programmlogik ausführt. Der Leser reagiert auf den Abstand des Chips etwa 3 cm von der Antenne.
  • Arduino vergleicht die Adresse der gelesenen Karte in einer Mehrfachbedingung, wendet digitale Zustände auf Dioden an, führt alle 100 ms einen Universalzähler aus (zählt in Zehntelsekunden) und steuert die Zählung beider Male.
  • Die Zeit wird über den I2C-Bus an die 20x4-Zeichenanzeige gesendet, wo sie aufgezeichnet wird. Das Display hält bis zu 30 ms bei einer I2C-Busgeschwindigkeit von 100 kHz - Standardgeschwindigkeit.
    • Maximaler Zeitbereich, der im System aufgelistet werden kann: 99 Stunden, 59 Minuten und 59 Sekunden
    Verbrauch: ca. 110mAh mit 5V Stromversorgung. Geeignet für die Stromversorgung über Power Bank, Ladegerät von einem Telefon mit USB-Ausgang.

#include <SPI.h>
#include <MFRC522.h>
#define RST_PIN         9
#define SS_1_PIN        10
#define NR_OF_READERS   1
byte ssPins[] = {SS_1_PIN};
MFRC522 mfrc522[NR_OF_READERS];
void setup() {
  Serial.begin(9600);
  while (!Serial);
  SPI.begin();
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    mfrc522[reader].PCD_Init(ssPins[reader], RST_PIN);
    mfrc522[reader].PCD_SetAntennaGain(112); // set to max (00001110)
    Serial.print(F("Reader "));
    Serial.print(reader);
    Serial.print(F(": "));
    mfrc522[reader].PCD_DumpVersionToSerial();
  }
}

void loop() {
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    if (mfrc522[reader].PICC_IsNewCardPresent() && mfrc522[reader].PICC_ReadCardSerial()) {
      Serial.print(F("Reader "));
      Serial.print(reader);
      Serial.print(F(": Card UID:"));
      dump_byte_array(mfrc522[reader].uid.uidByte, mfrc522[reader].uid.size);
      Serial.println();
      Serial.print(F("PICC type: "));
      MFRC522::PICC_Type piccType = mfrc522[reader].PICC_GetType(mfrc522[reader].uid.sak);
      Serial.println(mfrc522[reader].PICC_GetTypeName(piccType));
      mfrc522[reader].PICC_HaltA();
      mfrc522[reader].PCD_StopCrypto1();
    }
  }
}

void dump_byte_array(byte *buffer, byte bufferSize) {
  for (byte i = 0; i < bufferSize; i++) {
  }
  Serial.print("Kod:");
  unsigned long kod = 10000 * buffer[4] + 1000 * buffer[3] + 100 * buffer[2] + 10 * buffer[1] + buffer[0]; //finalny kod karty
  Serial.print(kod);
}

    Button DOMINATOR - Stoppuhr für Airsoft | Paintball

  • Verwendet die Tasteneingabe für Teams, Schiedsrichter und Radiergummi, insgesamt 4 Tasten.
  • Die Tasten ersetzen den RFID-Leser in der Eingabemethode für DOMINATOR.
  • Die Tastenverbindung befindet sich im INPUT_PULLUP-Modus. Durch Drücken von GND wird GND an den Arduino-Eingangsanschluss gebracht (Active-LOW-Signal).
  • Umschalttasten verwenden werden verwendet, keine Umschalttasten (sie würden nicht auf die Systemlogik reagieren, die gesamte Software würde immer "gedrückt" werden).
  • In der Basisversion reagiert DOMINATOR sofort auf eine Presse - die Zeitzählung für ein bestimmtes Team ist aktiviert (die Länge der Presse, die Anzahl der Pressen usw. werden nicht überprüft ...).
  • Die Zeiten werden auf einer 16x2- oder 20x4-LCD-Zeichenanzeige angezeigt
  • Systemlogik, Zähler ist identisch mit RFID DOMINATOR.
