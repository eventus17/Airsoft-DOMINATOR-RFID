Beware of RFID readers RC522 with fake (non-original NXP RC522) chip! - In the project on the UART it reports the chip version 0x12 - counterfeit (does not work for the DOMINATOR project, the original NXP chip with which the library cooperates is required!)

    • Project description:

  • Device - DOMINATOR includes a control microcontroller (Arduino Uno, Nano, or Mega 2560).
  • The LCD character display shows the times of the individual teams (RF and GDP team, respectively BLU and RED team.)
  • In the basic version of the software, there are 2 teams that play against each other and occupy this particular point!
  • At the beginning of the game, the display shows the times 00 hours, 00 minutes, 00 seconds for Team 1 (RED), Team 2, (BLUE).
  • Each member of the team is equipped with a card or key fob that communicates on the frequency 13.56MHz with an RFID reader.
  • Keychains can be color-coded (blue, red by team and feature ...)
  • If a team scores a point, any player on that team attaches their card to the reader, thereby awarding the point.
  • The diode of the given team (red / blue) will light up at this station and the time will start to be added to the given team (the change can also be announced by a buzzer - a detector, if it is used in the system).
  • If a point is occupied by the other team and a member of that team attaches his card, that team's LED will light up, the other team's time will be paused, and the time of the team that scored the point will be counted.
  • If the card / keychain is attached by the organizer (referee), both times will be paused until one of the teams introduces the card.
  • This type of card is also suitable for ending the game, when time stops and the winner is decided.
  • The last possible card in the system is to use the so-called eraser card that pauses and resets both times.
  • The project is suitable for Airsoft events and sports facilities.
  • The system is tested, functional, works smoothly, the ability to measure and display time in tenths of a second (programmatically counts but does not display, display in seconds).
  • The RC522 reader operates at a frequency of 13.56MHz, registers RFID tags at about 3 centimeters (almost physical contact with the reader is required), compatible ISO / IEC 14443 A tag format.

    • Hardware for the project - RFID DOMINATOR:

  • Arduino Uno/Nano
  • RFID reader RC522 (13.56MHz)
  • Standard keychains / cards / bracelets ISO/IEC 14443-A
  • Buzzer
  • 2x LED diodes

    • Principle of functionality on the hardware side:

  • The RF5 reader RC522 communicates with the Arduino via the SPI bus. It reads the UID (identifier) of RFID cards in range, forwards this identifier to Arduino, which executes the program logic. The reader responds to the distance of the chip about 3 cm from the antenna.
  • Arduino compares the read card address in a multi-condition, applies digital states to diodes, runs a universal counter every 100ms (counts in tenths of a second), controls the counting of both times.
  • The time is sent via the I2C bus to the 20x4 character display, where it is plotted. The display takes up to 30ms at I2C bus speed 100 kHz - Standard speed.
    • Maximum time range that can be listed in the system: 99 hours, 59 minutes and 59 seconds
    Consumption: approx. 110mAh with 5V power supply. Suitable to power via power bank, charger from a phone with USB output.

//http://arduino.clanweb.eu/airsoft-dominator.php?lang=en
//Autor: Bc. Martin Chlebovec (martinius96@gmail.com)

#include <SPI.h>
#include <MFRC522.h>
#define RST_PIN         9
#define SS_1_PIN        10
#define NR_OF_READERS   1
byte ssPins[] = {SS_1_PIN};
MFRC522 mfrc522[NR_OF_READERS];
void setup() {
  Serial.begin(9600);
  while (!Serial);
  SPI.begin();
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    mfrc522[reader].PCD_Init(ssPins[reader], RST_PIN);
    mfrc522[reader].PCD_SetAntennaGain(112); // set to max (00001110)
    Serial.print(F("Reader "));
    Serial.print(reader);
    Serial.print(F(": "));
    mfrc522[reader].PCD_DumpVersionToSerial();
  }
}

void loop() {
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    if (mfrc522[reader].PICC_IsNewCardPresent() && mfrc522[reader].PICC_ReadCardSerial()) {
      Serial.print(F("Reader "));
      Serial.print(reader);
      Serial.print(F(": Card UID:"));
      dump_byte_array(mfrc522[reader].uid.uidByte, mfrc522[reader].uid.size);
      Serial.println();
      Serial.print(F("PICC type: "));
      MFRC522::PICC_Type piccType = mfrc522[reader].PICC_GetType(mfrc522[reader].uid.sak);
      Serial.println(mfrc522[reader].PICC_GetTypeName(piccType));
      mfrc522[reader].PICC_HaltA();
      mfrc522[reader].PCD_StopCrypto1();
    }
  }
}

void dump_byte_array(byte *buffer, byte bufferSize) {
  for (byte i = 0; i < bufferSize; i++) {
  }
  Serial.print("Kod:");
  unsigned long kod = 10000 * buffer[4] + 1000 * buffer[3] + 100 * buffer[2] + 10 * buffer[1] + buffer[0]; //finalny kod karty
  Serial.print(kod);
}

    Button DOMINATOR - stopwatch for Airsoft | Paintball

  • Uses button input for teams, referee and eraser, a total of 4 buttons.
  • The buttons replace the RFID reader in the input method for DOMINATOR.
  • Button connection is in INPUT_PULLUP mode, pressing will bring GND to the Arduino input terminal (Active-LOW signal).
  • Use toggle buttons are used, not toggle buttons (they would not respond to system logic, all software would always be "pressed").
  • In the basic version, DOMINATOR responds immediately to a press - the time counting for the given team is activated (the length of the press, the number of presses, etc. are not verified ...).
  • Times are displayed on a 16x2 or 20x4 LCD character display
  • System logic, counter is identical to RFID DOMINATOR.
