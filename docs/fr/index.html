Description du projet:

DOMINATOR est un appareil - chronomètre pour les sports tels que Airsoft, Paintball. Le chronomètre utilise un microcontrôleur de contrôle Arduino (Arduino Uno / Nano) ou une puce séparée (Atmel) AtMega328P, qui contrôle l'ensemble de la logique du système. Le chronomètre comprend un écran LCD à caractères 16x2 (16 caractères x 2 lignes) ou 20x4, sur lequel les temps des équipes individuelles sont affichés. Il y a 2 équipes dans le jeu - équipe RED et équipe GRE qui jouent les uns contre les autres et occupent le point imaginaire formé par ce dispositif - DOMINATOR. Au début du jeu, l'écran affiche les heures 00 heures, 00 minutes, 00 secondes pour équipe RED , Équipe GRE . Chaque membre de l'équipe est équipé d'une carte, d'un bracelet ou d'un porte-clés, qui communique sur la fréquence 13,56 MHz avec un lecteur RFID, s'il est à portée. Les porte-clés et bracelets peuvent être codés par couleur (vert, rouge selon l'équipe et la fonction ...), ce qui permet de les distribuer plus efficacement aux joueurs, mais aussi leur archivage et stockage. Si l'équipe ROUGE occupe un point, le joueur de l'équipe attache sa carte au lecteur, occupant ainsi le point. Cette action déclenche un buzzer qui annonce un changement de point. Ensuite, la LED de l'équipe ROUGE à cette station s'allumera et l'écran commencera à ajouter du temps pour cette équipe. Si une équipe GRE occupe un point et qu'un membre de cette équipe attache la carte au lecteur, un buzzer retentit, la LED de cette équipe s'allume, le temps d'équipe de RED est mis en pause, et le temps d'équipe compte. GRE, qui a occupé le point. Si la carte / le porte-clés est attaché par l'organisateur (arbitre), les deux temps sont interrompus jusqu'à ce que l'une des équipes insère la carte. Ce type de carte convient donc pour entamer une pause, mais peut également être utilisée en fin de partie, lorsqu'il est nécessaire d'arrêter le temps pour évaluer la partie. Il existe encore un quatrième type de carte dans DOMINATOR - la gomme, qui fait une pause les deux fois et se réinitialise aux valeurs initiales comme après le démarrage du chronomètre DOMINATOR. Utilisé pour redémarrer le jeu, initialiser un nouveau jeu. Le projet convient aux événements Airsoft, aux installations sportives ou aux équipes Airsoft, qui peuvent construire l'équipement. L'avantage du système DOMINATOR est que le système peut être cloné et créer X points avec la même configuration via un code machine, qui peut être combattu indépendamment dans le jeu, tandis que les cartes RFID fonctionneront pleinement sur tous les appareils DOMINATOR. Le système est testé, fonctionnel, facile à entretenir. Le lecteur NXP RC522 fonctionne à une fréquence de 13,56 MHz, enregistre les étiquettes RFID à environ 3 centimètres (un contact presque physique avec le lecteur est nécessaire), format d'étiquette compatible ISO / IEC 14443-A. La norme d'étiquette donnée comprend également des cartes ISIC, bus qui fonctionnent comme des cartes NFC et peuvent communiquer leur adresse physique - UID à un lecteur à portée.

Matériel pour le projet - RFID DOMINATOR:

  • Arduino Uno/Nano
  • Lecteur RFID NXP RC522 (13,56 MHz)
  • Porte-clés / cartes / bracelets standard ISO / CEI 14443-A
  • Avertisseur sonore
  • 2x LED

    • Principe de fonctionnalité côté matériel:

  • Le lecteur RF5 RC522 communique avec l'Arduino via le bus SPI. Il lit l'UID (identifiant) des cartes RFID à portée, transmet cet identifiant à Arduino, qui exécute la logique du programme. Le lecteur répond à la distance de la puce à environ 3 cm de l'antenne.
  • Arduino compare l'adresse de la carte de lecture dans plusieurs conditions, applique des états numériques aux diodes, exécute un compteur universel toutes les 100 ms (compte en dixièmes de seconde), contrôle le comptage des deux temps.
  • L'heure est envoyée via le bus I2C à l'affichage de caractères 20x4, où elle est tracée. L'affichage prend jusqu'à 30 ms à la vitesse du bus I2C 100 kHz - Vitesse standard.
    • Plage de temps maximale pouvant être répertoriée dans le système: 99 heures, 59 minutes et 59 secondes
    Consommation: environ 110mAh avec alimentation 5V. Pratique pour alimenter via batterie externe, chargeur à partir d'un téléphone avec sortie USB.

DOMINATOR convient également aux équipes, aux installations sportives, qui peuvent utiliser cet appareil pour étendre les modes de jeu proposés pour les jeux individuels. DOMINATOR convient aux modes de jeu avec le même nombre de joueurs dans les deux équipes, par ex. DOMINATION, Capture Point (plus d'un DOMINATOR peut être utilisé dans le jeu). /*|Titre: Chargement des cartes RFID UID                     |*/

/*|UID sur l'interface UART.                                 |*/
/*|Les cartes UID enregistrées sont utilisées dans DOMINATOR |*/

/*|E-mail: martinius96@gmail.com                             |*/

#include <SPI.h>
#include <MFRC522.h>
#define RST_PIN         9
#define SS_1_PIN        10
#define NR_OF_READERS   1
byte ssPins[] = {SS_1_PIN};
MFRC522 mfrc522[NR_OF_READERS];
void setup() {
  Serial.begin(9600);
  while (!Serial);
  SPI.begin();
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    mfrc522[reader].PCD_Init(ssPins[reader], RST_PIN);
    mfrc522[reader].PCD_SetAntennaGain(112); // set to max (00001110)
    Serial.print(F("Reader "));
    Serial.print(reader);
    Serial.print(F(": "));
    mfrc522[reader].PCD_DumpVersionToSerial();
  }
}

void loop() {
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    if (mfrc522[reader].PICC_IsNewCardPresent() && mfrc522[reader].PICC_ReadCardSerial()) {
      Serial.print(F("Reader "));
      Serial.print(reader);
      Serial.print(F(": Card UID:"));
      dump_byte_array(mfrc522[reader].uid.uidByte, mfrc522[reader].uid.size);
      Serial.println();
      Serial.print(F("PICC type: "));
      MFRC522::PICC_Type piccType = mfrc522[reader].PICC_GetType(mfrc522[reader].uid.sak);
      Serial.println(mfrc522[reader].PICC_GetTypeName(piccType));
      mfrc522[reader].PICC_HaltA();
      mfrc522[reader].PCD_StopCrypto1();
    }
  }
}

void dump_byte_array(byte *buffer, byte bufferSize) {
  for (byte i = 0; i < bufferSize; i++) {
  }
  Serial.print("Code charge: ");
  unsigned long kod = 10000 * buffer[4] + 1000 * buffer[3] + 100 * buffer[2] + 10 * buffer[1] + buffer[0]; //finalny kod karty
  Serial.println(kod);
}

    Scanner I2C - un programme pour lire l'adresse du convertisseur I2C pour l'écran LCD

    Sortie attendue du scanner I2C:

    La bibliothèque Wire est intégrée à l'IDE Arduino, aucune installation n'est requise. Le programme peut être téléchargé immédiatement. Il est conseillé de connecter le matériel - convertisseur I2C pour un écran 16x2 / 20x4, même sans écran, avant de démarrer le programme. 
    #include <Wire.h>
 
 
void setup()
{
  Wire.begin();
 
  Serial.begin(9600);
  while (!Serial);             // Leonardo: wait for serial monitor
  Serial.println("\nI2C Scanner");
}
 
 
void loop()
{
  byte error, address;
  int nDevices;
 
  Serial.println("Scanning...");
 
  nDevices = 0;
  for(address = 1; address < 127; address++ )
  {
    // The i2c_scanner uses the return value of
    // the Write.endTransmisstion to see if
    // a device did acknowledge to the address.
    Wire.beginTransmission(address);
    error = Wire.endTransmission();
 
    if (error == 0)
    {
      Serial.print("I2C device found at address 0x");
      if (address < 16)
        Serial.print("0");
      Serial.print(address,HEX);
      Serial.println("  !");
 
      nDevices++;
    }
    else if (error==4)
    {
      Serial.print("Unknown error at address 0x");
      if (address < 16)
        Serial.print("0");
      Serial.println(address,HEX);
    }    
  }
  if (nDevices == 0)
    Serial.println("No I2C devices found\n");
  else
    Serial.println("done\n");
 
  delay(5000);           // wait 5 seconds for next scan
}

    Button DOMINATOR est un appareil - chronomètre pour les sports tels que Airsoft, Paintball. Le chronomètre utilise un microcontrôleur de contrôle Arduino (Arduino Uno / Nano) ou une puce séparée (Atmel) AtMega328P, qui contrôle l'ensemble de la logique du système. Button DOMINATOR utilise l'entrée de bouton pour l'action d'occuper un point par un membre de l'équipe, ainsi que pour l'entrée d'un arbitre et d'une entité de gomme. Ainsi, un total de 4 boutons pour 4 entités système, les boutons de commutation connectés en mode INPUT_PULLUP sont utilisés. Appuyez sur le bouton pour envoyer le signal GND à la borne d'entrée numérique Arduino (signal de commutation Active-LOW). Les boutons de commutation sont utilisés, pas les boutons de commutation (ils ne réagiraient pas à la logique du système, tous les logiciels seraient toujours «enfoncés»). Dans la version de base, DOMINATOR répond immédiatement à une presse - le comptage du temps pour une équipe donnée est activé (la longueur de la presse, le nombre de pressions, etc. ne sont pas vérifiés ...). Les heures sont affichées sur un écran LCD à caractères de 16x2 ou 20x4, qui communique avec le microcontrôleur de contrôle via le bus I2C. Si un membre de l'équipe RED occupe un point en appuyant sur un bouton, un buzzer se déclenche, qui annonce un changement de point avec un bip court. Ensuite, la LED de l'équipe ROUGE à cette station s'allumera et l'écran commencera à ajouter du temps pour cette équipe. Si le point est occupé par l'équipe GRE et qu'un membre de l'équipe appuie sur le bouton approprié, le buzzer retentit, la LED GRE team s'allume, le temps de l'équipe ROUGE est mis en pause et le Le temps de l'équipe GRE est calculé quel point il a occupé. Si l'organisateur (arbitre) appuie sur son bouton, les deux temps sont suspendus jusqu'à ce qu'un membre de l'une des équipes fasse une entrée utilisateur. Ce type d'entrée est donc adapté pour démarrer une pause, mais peut également terminer le jeu pour évaluation. Bouton DOMINATOR a un quatrième bouton pour l'entité gomme, qui s'arrête et réinitialise les deux fois aux valeurs initiales comme après le démarrage du chronomètre Button DOMINATOR. Utilisé pour redémarrer le jeu, initialiser un nouveau jeu. Le projet convient aux événements Airsoft, aux installations sportives ou aux équipes Airsoft, qui peuvent construire l'équipement. L'avantage du système Button DOMINATOR est qu'il peut être cloné et créer des points X avec la même configuration via un code machine, qui peut être chargé dans un nombre illimité de cartes et puces Arduino et garantit un fonctionnement d'application identique. Les joueurs peuvent ainsi marquer plus de points dans la partie, tandis qu'à la fin de la partie, le temps est additionné tant que le point était occupé par quelle équipe.

    Programme de test (Shareware) - Bouton DOMINATOR avec code machine

    Une version Shareware (.hex) est disponible pour le projet Button DOMINATOR. La version vous permet de mesurer un maximum de 15 secondes de jeu pour chaque équipe. L'objectif de la version est de vérifier la compatibilité avec votre matériel. Les codes machine basés sur l'adresse I2C du convertisseur LCD sont disponibles dans le référentiel Github. Les adresses les plus utilisées sont 0x27 et 0x3F. Si vous ne connaissez pas l'adresse de votre convertisseur, utilisez l'exemple intégré dans Arduino IDE -> I2C Scanner, qui écrira l'adresse sur l'interface UART (moniteur série). Le code machine peut être téléchargé sur le microcontrôleur Arduino Uno via le logiciel AVRDUDE - qui fait partie de l'IDE Arduino, ou avec une interface graphique via XLoader - plus facile à utiliser.

  • AVRDUDE doit être exécuté via le CMD (ligne de commande) installé avec l'IDE Arduino. Situé dans C: /Users/User/Documents/ArduinoData/packages/arduino/tools/avrdude/6.X.X-arduino/bin
    • Exemple de commande pour télécharger un programme sur Arduino Uno / Nano (AtMega328P):
  • avrdude -CC: avrdude.conf -v -patmega328p -carduino -PCOM21 -b115200 -D -Uflash: w: sketch.ino.hex: i
  • XLoader est une application Windows (outil exécutable .exe) avec une interface graphique simple , qui permet la sélection du firmware - code machine (.hex), la plate-forme cible et peut afficher dynamiquement le COM port sur lequel se trouve l'appareil.
  • En cliquant simplement sur Télécharger, le code machine peut être téléchargé automatiquement (XLoader utilise l'outil AVRDUDE similaire à l'IDE Arduino). La fenêtre XLoader informe sur le téléchargement réussi sous la forme d'une liste d'octets téléchargés.
    • Le micrologiciel peut être téléchargé depuis le référentiel du projet: ICI . Le firmware se trouve dans le dossier / firmware / divisé en fonction de l'adresse du convertisseur I2C (que vous avez découvert à partir de la sortie du scanner I2C sur l'interface UART).
