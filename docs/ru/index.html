Остерегайтесь считывателей RFID RC522 с поддельным (не оригинальным NXP RC522) чипом! - В проекте на UART сообщает версия чипа 0x12 - контрафакт (не работает для проекта DOMINATOR, требуется оригинальный чип NXP, с которым взаимодействует библиотека!)

    • Описание проекта:

  • Устройство - DOMINATOR включает управляющий микроконтроллер (Arduino Uno, Nano или Mega 2560).
  • Символьный ЖК-дисплей показывает время отдельных команд (команды РФ и ВВП, соответственно команды СИН и КРАСНЫХ).
  • В базовой версии программы есть 2 команды, которые играют друг против друга и занимают именно эту точку!
  • В начале игры на дисплее отображается время 00 часов, 00 минут, 00 секунд для Команды 1 (КРАСНЫЙ), Команды 2 (СИНИЙ).
  • У каждого члена команды есть карта или брелок, который на частоте 13,56 МГц обменивается данными с RFID-считывателем.
  • Брелки могут иметь цветовую кодировку (синий, красный для команды и функции ...)
  • Если команда набирает очко, любой игрок этой команды прикрепляет свою карту к считывателю, тем самым присуждая очко.
  • На этой станции загорится диод заданной команды (красный / синий) и время начнет добавляться к заданной команде (изменение также может быть объявлено зуммером - детектором, если он используется в системе).
  • Если очко занято другой командой и член этой команды прикрепляет свою карточку, загорается светодиод команды, время другой команды приостанавливается, и учитывается время команды, занявшей очко.
  • Если карта / брелок прикреплен организатором (судьей), оба раза будут приостановлены, пока одна из команд не предъявит карту.
  • Этот тип карт также подходит для завершения игры, когда время останавливается и определяется победитель.
  • Последняя возможная карта в системе - использовать так называемую ластик, который приостанавливает и сбрасывает оба раза.
  • Проект подходит для страйкбольных мероприятий и спортивных сооружений.
  • Система протестирована, работоспособна, работает без сбоев, возможность измерять и отображать время в десятых долях секунды (программно считает, но не отображает, отображение в секундах).
  • Считыватель RC522 работает на частоте 13,56 МГц, регистрирует RFID-метки на расстоянии около 3 сантиметров (требуется почти физический контакт со считывателем), совместим с форматом метки ISO / IEC 14443 A.

    • Оборудование для проекта - RFID DOMINATOR:

  • Arduino Uno/Nano
  • RFID читатель RC522 (13.56MHz)
  • Стандартные брелки / карточки / браслеты ISO / IEC 14443-A
  • Детектор (пассивный зуммер)
  • 2x светодиода

    • Принцип работы оборудования:

  • Считыватель RF5 RC522 связывается с Arduino через шину SPI. Он считывает UID (идентификатор) RFID-карт в диапазоне, пересылает этот идентификатор в Arduino, который выполняет логику программы. Считыватель реагирует на расстояние чипа примерно в 3 см от антенны.
  • Arduino сравнивает адрес считываемой карты в нескольких условиях, применяет цифровые состояния к диодам, запускает универсальный счетчик каждые 100 мс (счет в десятых долях секунды), контролирует счет обоих времен.
  • Время отправляется по шине I2C на символьный дисплей 20x4, где оно отображается. Дисплей работает до 30 мс при скорости шины I2C 100 кГц - стандартная скорость.
    • Максимальный временной диапазон, который может быть указан в системе: 99 часов 59 минут 59 секунд
    Потребление: около 110 мАч при питании 5 В. Подходит для питания через power bank, зарядное устройство от телефона с выходом USB.

#include <SPI.h>
#include <MFRC522.h>
#define RST_PIN         9
#define SS_1_PIN        10
#define NR_OF_READERS   1
byte ssPins[] = {SS_1_PIN};
MFRC522 mfrc522[NR_OF_READERS];
void setup() {
  Serial.begin(9600);
  while (!Serial);
  SPI.begin();
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    mfrc522[reader].PCD_Init(ssPins[reader], RST_PIN);
    mfrc522[reader].PCD_SetAntennaGain(112); // set to max (00001110)
    Serial.print(F("Reader "));
    Serial.print(reader);
    Serial.print(F(": "));
    mfrc522[reader].PCD_DumpVersionToSerial();
  }
}

void loop() {
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    if (mfrc522[reader].PICC_IsNewCardPresent() && mfrc522[reader].PICC_ReadCardSerial()) {
      Serial.print(F("Reader "));
      Serial.print(reader);
      Serial.print(F(": Card UID:"));
      dump_byte_array(mfrc522[reader].uid.uidByte, mfrc522[reader].uid.size);
      Serial.println();
      Serial.print(F("PICC type: "));
      MFRC522::PICC_Type piccType = mfrc522[reader].PICC_GetType(mfrc522[reader].uid.sak);
      Serial.println(mfrc522[reader].PICC_GetTypeName(piccType));
      mfrc522[reader].PICC_HaltA();
      mfrc522[reader].PCD_StopCrypto1();
    }
  }
}

void dump_byte_array(byte *buffer, byte bufferSize) {
  for (byte i = 0; i < bufferSize; i++) {
  }
  Serial.print("Kod:");
  unsigned long kod = 10000 * buffer[4] + 1000 * buffer[3] + 100 * buffer[2] + 10 * buffer[1] + buffer[0]; //finalny kod karty
  Serial.print(kod);
}

    Кнопка DOMINATOR - секундомер для страйкбола | Пейнтбол

  • Использует кнопки для команд, рефери и ластик, всего 4 кнопки.
  • Кнопки заменяют считыватель RFID в методе ввода для DOMINATOR.
  • Кнопка подключения находится в режиме INPUT_PULLUP, нажатие приведет к GND на входной клемме Arduino (сигнал Active-LOW).
  • Используются кнопки питания, а не кнопки (они не будут реагировать на системную логику, все программное обеспечение всегда будет "нажато").
  • В базовой версии DOMINATOR немедленно реагирует на нажатие - активирован счет времени для данной команды (продолжительность нажатия, количество нажатий и т. д. не проверяются ...).
  • Время отображается на ЖК-дисплее размером 16 x 2 или 20 x 4.
  • Системная логика, счетчик идентичен RFID DOMINATOR.
