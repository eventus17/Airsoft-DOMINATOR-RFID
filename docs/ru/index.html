RFID DOMINATOR - stopky pre Airsoft | Paintball

Остерегайтесь считывателей RFID RC522 с поддельным (неоригинальным NXP RC522) чипом! - В проекте по UART сообщает: Версия прошивки: 0x12 - фальшивая микросхема. Проект с фейковым чипом не работает. Протестировано и работает с считывателем RFID NXP RC522, который сообщает версию прошивки: версия прошивки 0x92 (версия 2.0).

Описание проекта:

  • Устройство - DOMINATOR - секундомер для страйкбола и пейнтбола, который позволяет рассчитывать и визуализировать время для двух команд.
  • Они используют микроконтроллер управления Arduino (Arduino Uno / Nano).
  • ЖК-символы 16x2 и 20x4 отображают время отдельных команд (RED и GRE).
  • В системе есть 2 команды, которые играют друг против друга и занимают конкретную точку - DOMINATOR.
  • В начале игры на дисплее отображается время 00 часов, 00 минут, 00 секунд для Команды 1 (КРАСНЫЙ), Команды 2 (ЗЕЛЕНЫЙ).
  • У каждого члена команды есть карта или брелок, который на частоте 13,56 МГц обменивается данными с RFID-считывателем.
  • Брелки могут иметь цветовую кодировку (синий, красный для команды и функции ...), что позволяет более эффективно распределять их среди игроков.
  • Если команда набирает очко, игрок этой команды прикрепляет свою карту к считывателю, тем самым присуждая очко.
  • Светодиод команды (красный / зеленый) на этой станции загорится, и время будет добавлено к команде (изменение также может быть объявлено зуммером - детектором, если он используется в системе).
  • Если очко занято другой командой и член этой команды прикрепляет свою карточку, загорается светодиод этой команды, время другой команды будет приостановлено, и будет засчитано время команды, набравшей очко. .
  • Если карта / брелок прикреплен организатором (судьей), оба раза будут приостановлены, пока одна из команд не предъявит карту.
  • Этот тип карты также подходит для перерыва, окончания игры, когда время останавливается и определяется победитель.
  • Последняя возможная карта в системе - использовать так называемую ластик, который приостанавливает и сбрасывает оба раза.
  • Проект подходит для мероприятий по страйкболу, спортивных сооружений или команд по страйкболу.
  • Система проверена, работоспособна, работает бесперебойно, возможность измерения и отображения времени даже в десятых долях секунды (они программно считают, но не отображают, в базовой версии отображаются с разрешением секунд).
  • Считыватель RC522 работает на частоте 13,56 МГц, регистрирует RFID-метки на расстоянии около 3 сантиметров (требуется почти физический контакт со считывателем), совместим с форматом меток ISO / IEC 14443-A.
  • Считыватель RFID NXP RC522 также регистрирует карты ATM, ISIC, bus, которые действуют как карты NFC и могут передавать свой физический адрес считывающему устройству в пределах досягаемости.

    • Принцип работы аппаратной части:

  • Считыватель RF5 RC522 связывается с Arduino через шину SPI. Он считывает UID (идентификатор) RFID-карт в диапазоне, пересылает этот идентификатор в Arduino, который выполняет логику программы. Считыватель реагирует на расстояние чипа примерно в 3 см от антенны.
  • Arduino сравнивает адрес считываемой карты в нескольких условиях, применяет цифровые состояния к диодам, запускает универсальный счетчик каждые 100 мс (счет в десятых долях секунды), контролирует счет обоих времен.
  • Время отправляется по шине I2C на символьный дисплей 20x4, где оно отображается. Дисплей работает до 30 мс при скорости шины I2C 100 кГц - стандартная скорость.
    • Максимальный диапазон времени, который может быть указан в системе: 99 часов 59 минут и 59 секунд.
    Потребление: около 110 мАч при питании 5 В. Удобно получать питание через внешний аккумулятор, зарядное устройство от телефона с выходом USB.

    Программа для чтения UID адресов RFID карт для DOMINATOR

  • Программа используется для получения идентификаторов (UID) RFID-тегов, карт, которые будут использоваться в программе для RFID DOMINATOR
  • Адрес карты указан в последовательном мониторе (UART) - используется 9600 бод / с.
  • Рекомендуется сохранить и перезаписать отчет в виде чистого документа .xls (EXCEL), что упростит управление назначенными RFID-картами для определенных групп и организаций.
    • Вы должны импортировать библиотеку MFRC522 в вашу среду разработки Arduino, чтобы успешно загрузить исходный код для загрузки карт UID. Загрузите репозиторий проекта RFID DOMINATOR (на который указывает первая ссылка вверху страницы), скопируйте содержимое папки /src/ (папка MFRC522) в папку / Arduino / libraries / (путь к папке зависит от того, где у вас установлена IDE Arduino), чаще всего в Documents / Program Files в более старых операционных системах Windows.

/*| Имя: Загрузка карт UID RFID                      |*/
/*| Описание: Считывает через считыватель RFID RC522 |*/
/*| Адрес RFID-карты, которая прикреплена и печатает |*/
/*| UID на интерфейсе UART.                          |*/
/*| Записанные карты UID используются в DOMINATOR    |*/
/*|Autor: Martin Chlebovec                           |*/
/*|E-mail: martinius96@gmail.com                     |*/
/*|Licencia pouzitia: MIT                            |*/

#include <SPI.h>
#include <MFRC522.h>
#define RST_PIN         9
#define SS_1_PIN        10
#define NR_OF_READERS   1
byte ssPins[] = {SS_1_PIN};
MFRC522 mfrc522[NR_OF_READERS];
void setup() {
  Serial.begin(9600);
  while (!Serial);
  SPI.begin();
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    mfrc522[reader].PCD_Init(ssPins[reader], RST_PIN);
    mfrc522[reader].PCD_SetAntennaGain(112); // set to max (00001110)
    Serial.print(F("Reader "));
    Serial.print(reader);
    Serial.print(F(": "));
    mfrc522[reader].PCD_DumpVersionToSerial();
  }
}

void loop() {
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    if (mfrc522[reader].PICC_IsNewCardPresent() && mfrc522[reader].PICC_ReadCardSerial()) {
      Serial.print(F("Reader "));
      Serial.print(reader);
      Serial.print(F(": Card UID:"));
      dump_byte_array(mfrc522[reader].uid.uidByte, mfrc522[reader].uid.size);
      Serial.println();
      Serial.print(F("PICC type: "));
      MFRC522::PICC_Type piccType = mfrc522[reader].PICC_GetType(mfrc522[reader].uid.sak);
      Serial.println(mfrc522[reader].PICC_GetTypeName(piccType));
      mfrc522[reader].PICC_HaltA();
      mfrc522[reader].PCD_StopCrypto1();
    }
  }
}

void dump_byte_array(byte *buffer, byte bufferSize) {
  for (byte i = 0; i < bufferSize; i++) {
  }
  Serial.print("Code: ");
  unsigned long kod = 10000 * buffer[4] + 1000 * buffer[3] + 100 * buffer[2] + 10 * buffer[1] + buffer[0]; //finalny kod karty
  Serial.println(kod);
}

    I2C Scanner - программа для чтения адреса преобразователя I2C для ЖК-дисплея

    Ожидаемый результат I2C Scanner:

    Библиотека Wire встроена в IDE Arduino, установка не требуется. Программа может быть загружена сразу. Перед запуском программы желательно подключить аппаратную часть - преобразователь I2C для дисплея 16x2 / 20x4, даже без дисплея. 
    #include <Wire.h>
 
 
void setup()
{
  Wire.begin();
 
  Serial.begin(9600);
  while (!Serial);             // Leonardo: wait for serial monitor
  Serial.println("\nI2C Scanner");
}
 
 
void loop()
{
  byte error, address;
  int nDevices;
 
  Serial.println("Scanning...");
 
  nDevices = 0;
  for(address = 1; address < 127; address++ )
  {
    // The i2c_scanner uses the return value of
    // the Write.endTransmisstion to see if
    // a device did acknowledge to the address.
    Wire.beginTransmission(address);
    error = Wire.endTransmission();
 
    if (error == 0)
    {
      Serial.print("I2C device found at address 0x");
      if (address < 16)
        Serial.print("0");
      Serial.print(address,HEX);
      Serial.println("  !");
 
      nDevices++;
    }
    else if (error==4)
    {
      Serial.print("Unknown error at address 0x");
      if (address < 16)
        Serial.print("0");
      Serial.println(address,HEX);
    }    
  }
  if (nDevices == 0)
    Serial.println("No I2C devices found\n");
  else
    Serial.println("done\n");
 
  delay(5000);           // wait 5 seconds for next scan
}

    Кнопка DOMINATOR - секундомер для страйкбола | Пейнтбол

  • Использует кнопочный ввод для команд, судью и ластик, всего 4 кнопки с функцией переключения.
  • Кнопки заменяют считыватель RFID в методе ввода для DOMINATOR.
  • Кнопка подключения находится в режиме INPUT_PULLUP, нажатие приведет к подключению GND к входному разъему Arduino (сигнал Active-LOW).
  • Используются кнопки-переключатели, а не кнопки-переключатели (они не будут реагировать на системную логику, все программное обеспечение всегда будет "нажато").
  • В базовой версии DOMINATOR немедленно реагирует на нажатие - активирован счет времени для данной команды (продолжительность нажатия, количество нажатий и т. д. не проверяются ...).
  • Время отображается на ЖК-дисплее размером 16 x 2 или 20 x 4.
  • Системная логика, счетчик идентичен RFID DOMINATOR.

  • зуммер

    Тестовая программа (Условно-бесплатная) - Кнопка ДОМИНАТОР с машинным кодом

    Условно-бесплатная версия (.hex) доступна для проекта Button DOMINATOR. Эта версия позволяет измерять максимум 15 секунд игры для каждой команды. Цель версии - проверить совместимость с вашим оборудованием. Машинные коды на основе адреса преобразователя I2C для ЖК-дисплея доступны в репозитории Github. Наиболее часто используемые адреса - 0x27 и 0x3F. Если вы не знаете адрес вашего конвертера, используйте встроенный пример в Arduino IDE -> I2C Scanner, который запишет адрес в интерфейс UART (Serial monitor). Машинный код может быть загружен в микроконтроллер Arduino Uno через программное обеспечение AVRDUDE - часть Arduino IDE или с графическим интерфейсом через XLoader - проще в использовании.

  • AVRDUDE должен запускаться через CMD (командную строку), установленную с Arduino IDE. Находится в C: /Users/User/Documents/ArduinoData/packages/arduino/tools/avrdude/6.X.X-arduino/bin
    • Пример команды для загрузки программы в Arduino Uno / Nano (AtMega328P):
  • avrdude -CC: avrdude.conf -v -patmega328p -carduino -PCOM21 -b115200 -D -Uflash: w: sketch.ino.hex: i
  • XLoader - это приложение Windows (исполняемый инструмент .exe) с простым графическим интерфейсом , которое позволяет вам выбирать прошивку - машинный код (.hex), целевую платформу и может динамически отображать COM-порт, на котором находится устройство.
  • Если просто щелкнуть «Загрузить», машинный код может быть загружен автоматически (XLoader использует инструмент AVRDUDE, аналогичный Arduino IDE). Окно XLoader сообщает об успешной загрузке в виде списка загруженных байтов.
    • Прошивку можно скачать из репозитория проекта: здесь. Прошивка находится в папке / firmware /, разделенной по адресу преобразователя I2C (вы узнали из вывода сканера I2C на интерфейс UART).
